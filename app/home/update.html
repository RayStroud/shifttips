Updates / Changes

Wed Mar 15, 2017

Added a Location value to track shifts at different restaurants. If there are any troubles, please Email the Administrator. The new features include:

  • • The Add Shift page, Shift Details page, Edit Shift page, and Shifts Grid page have a new location field. Read more in the "What do all the fields mean?" section on the About / Help / FAQ page.
  • • The Location field can be enabled or disabled on the Account page. Read more in the "How to use the device view settings?" section on the About / Help / FAQ page.
  • • The summary page now has an option to view shift summaries grouped by location.
  • • The default values for Wage and Location have been moved to their own section on the Account page. Read more in the "How to set default values?" section on the About / Help / FAQ page.
  • • The device view settings are stored differently behind the scenes. This might cause some issues if the app fails to properly reset. If there are any issues, please Email the Administrator.

Mon Feb 20, 2017

The server was moved and reconfigured. If there are any issues, please Email the Administrator.

Tue Nov 8, 2016

The login process has changed behind-the-scenes. If there are any issues, please Email the Administrator.

Tue Oct 25, 2016

On the summary and period summary pages, there is an option to see either the averages or totals or both. Read more in the "How to use the summaries?" section on the About / Help / FAQ page.

Thu Oct 20, 2016

Finally updated the app with a lot of features that have been building up. If there are any troubles, please Email the Administrator. The old version is also online, if needed. The new features include:

  • • The summary page now has an option to view shift summaries grouped by length (in half-hours).
  • • The text-size option was added to the Filters bar. Read more in the "How to use the filters?" section on the About / Help / FAQ page.
  • • The view settings on the Account page are now collapsible for easier access on mobile devices. Users can save the current view settings, which can be reloaded on any device. Read more in the "How to use the device view settings?" section on the About / Help / FAQ page.
  • • The filters and device view settings are stored differently behind-the-scenes. Hopefully this doesn't cause any problems.
  • • The login process has changed behind-the-scenes, which will likely cause the app to prompt for a re-login.
